### Changelog

##### 1.0.13

* __Breaking changes__: Методы All и ById интерфейса IQueryFor<> теперь являются extension-методами
* __Breaking changes__: Пространство имен ByndyuSoft.Infrastructure.Domain.Criterion переименовано в ByndyuSoft.Infrastructure.Domain.Criteria в соответствии с правилами английского языка
* Добавлен extension-метод Count() для IQueryBuilder для выполнения запросов-подсчетов
* Добавлен абстрактный класс LinqQueryBase с 2мя generic-аргументами для выполнения мультитабличных запросов
* Добавлен extension-метод DistinctBy() для интерфейса IEnumerable<> для выборки уникальных по какому-либо свойству объектов
* Добавлны методы Contains() и LikewiseContains() для строк, позволяющие выполнить проверку на вхождения подстроки в строку и возвращающие bool
* Добавлено перечисление SortOrder, которое может быть использовано при организации сортировок в клиентском коде
* Добавлен extension-метод Invert() для перечисления SortOrder, позволяющий получить направление сортировки обратное текущему
* Добавлен интерфейс IEntityComponent, являющийся маркировочным для классов, чьи экземпляры являются компонентами сущностей
* Добавлен базовый класс конфигурации автомаппингов FluetnNHibernate'а, учитывающий наличие интерфейса IEntityComponent
* Добавлен DotSettings-файл настроек ReSharper'а специфичный для solution'а
* Небольшие реорганизации кода и файлов в репозитории
* Начал вестись changelog всех изменений начиная с предыдущей версии

##### 1.0.12

* Добавили метод IQueryFor<>.All и критерион AllEntities в сборку Common
* Добавили extension-метод AsArray() для интерфейса IEnumerable<>
* Обновили шаблонный пример на ASP.NET MVC
* Перешли на NUnit, т.к. он везде поддерживается нативно
